📘 UI Rules (MVP)

docs/03_ui_rules.md

1. 목적 (Purpose)

본 문서는 MasterPlan MVP UI 설계 규칙을 정의한다.

제조 프로젝트 특성에 맞는

단순하고

예측 가능한 UI

를 목표로 한다.

2. 기본 UI 구조
[ 프로젝트 선택 ]
        ↓
[ 분류 트리 (좌측) ]  |  [ 선택된 분류 상세/목록 (우측) ]

3. 분류 트리 UI 규칙
3.1 트리 구조

depth=0 (대분류): 항상 펼친 상태

depth=1 (중분류): 기본 펼침

depth=2 (소분류): 리프 노드 (children 없음)

3.2 트리 표시 규칙
depth	UI 표현
0	Bold / 아이콘 표시
1	일반 텍스트
2	리스트 항목
3.3 정렬 규칙

sort_no ASC

name ASC

백엔드 정렬 결과를 그대로 신뢰하며, 프론트에서는 재정렬하지 않는다.

4. 사용자 인터랙션 규칙
4.1 클릭 동작

대/중분류 클릭

트리 펼침/접힘

소분류 클릭

우측 패널에 해당 소분류의 작업 목록 표시

4.2 선택 상태

선택된 분류는 강조 표시

URL에 path 또는 classification_id 반영 (선택)

5. MVP UI 제한 사항

드래그 & 드롭 ❌

분류 순서 변경 ❌

분류 생성/수정/삭제 UI ❌

depth 4 이상 표현 ❌

6. 에러/빈 상태 처리
6.1 데이터 없음

“등록된 분류가 없습니다.”

6.2 API 에러

토스트 또는 상단 알림

상세 에러 메시지는 MVP에서 숨김

7. 디자인 원칙 요약

❌ 과도한 기능

❌ 복잡한 트리 편집

✅ 읽기 쉬운 구조

✅ 제조 공정 흐름에 집중
